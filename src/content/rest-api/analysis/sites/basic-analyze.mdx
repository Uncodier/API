# Basic Analysis API (`/api/analyze`)

This API provides a simple way to analyze a website via a POST request.

## Features

- Simple and straightforward implementation
- Uses the `analyzeSiteAction` function to perform the analysis
- Focused on identifying site sections (hero, features, testimonials, etc.)
- Basic error handling with Zod schema validation

## Endpoint

```
POST /api/analyze
```

## Request Parameters

| Parameter | Type   | Description                   | Required |
|-----------|--------|-------------------------------|----------|
| url       | string | Website URL to analyze        | Yes      |
| includeRawHtml | boolean | Include raw HTML in the response | No (default: false) |

## Example Request

```json
{
  "url": "https://example.com",
  "includeRawHtml": false
}
```

## Response

The response includes either the analysis result or an error message.

### Success Response

```json
{
  "success": true,
  "url": "https://example.com",
  "analysis": {
    // Site analysis data
    // If includeRawHtml is true, it will include:
    "rawHtml": "<!DOCTYPE html><html>...</html>"
  }
}
```

### Error Response

```json
{
  "success": false,
  "error": "Error message"
}
```

## Important Notes

> Note: This API is a simpler implementation compared to the advanced `/api/site/analyze` API. If you need more advanced options, consider using the advanced API.

## Differences with the Advanced API

This API (`/api/analyze`) differs from the advanced API (`/api/site/analyze`) in the following ways:

1. Implementation: Uses the `analyzeSiteAction` function instead of specialized services.
2. Options: Does not allow configuring advanced options such as analysis type, depth, timeout, etc.
3. Response: Returns a simpler response structure.
4. Error handling: Implements a more basic error handling system.

For more advanced use cases, we recommend using the advanced API `/api/site/analyze`.
