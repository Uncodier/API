# List Available Integrations

Retrieve a list of all available integrations that can be connected to agents.

## Endpoint

```http
GET /api/agents/integrations
```

## Query Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `category` | string | No | Filter integrations by category (e.g., "analytics", "crm", "communication") |
| `status` | string | No | Filter integrations by status ("active", "beta", "deprecated") |

## Response

### Success Response

```json
{
  "success": true,
  "data": {
    "integrations": [
      {
        "id": "integration_123",
        "name": "Google Analytics",
        "description": "Connect with Google Analytics to track website metrics",
        "category": "analytics",
        "status": "active",
        "version": "1.0.0",
        "provider": "Google",
        "authentication": {
          "type": "oauth2",
          "scopes": [
            "https://www.googleapis.com/auth/analytics.readonly"
          ]
        },
        "capabilities": [
          "read_analytics",
          "track_events",
          "generate_reports"
        ],
        "configuration": {
          "required_fields": [
            "client_id",
            "client_secret",
            "redirect_uri"
          ],
          "optional_fields": [
            "view_id"
          ]
        }
      }
    ],
    "total": 1,
    "page": 1,
    "per_page": 10
  }
}
```

### Error Response

```json
{
  "success": false,
  "error": {
    "code": "INVALID_FILTER",
    "message": "Invalid filter parameters provided"
  }
}
```

## Error Codes

| Code | Description |
|------|-------------|
| `INVALID_FILTER` | The provided filter parameters are invalid |
| `SERVER_ERROR` | An error occurred while retrieving the integrations |

## Notes

- Integrations are paginated with 10 items per page by default
- Use the `category` parameter to filter integrations by their primary function
- Use the `status` parameter to filter integrations by their current status
- Each integration includes its authentication requirements and capabilities
- OAuth2 integrations require proper client credentials and scopes
- Some integrations may require additional configuration after authentication
